Module: JobInformation

The fetchJobs POST endpoint allows for a user to fetch a list of recommended jobs. In the function, both the job id and user id are passed into the endpoint. This allows the backend to retrieve from the database collection user information and job listing information. The system utilizes a querying function provided by Firestore for fast lookup. In this manner the top K (10 in the final design) tags can be extracted from each job in the job listing database, and a ranking can be established with the tags tied to the user making the query.
